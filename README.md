## 博客改版计划

新技术说明

> 前言: 总之,之前选择github.io做博客原因在于自由度高,虽然不支持互动评论等,但是页面简洁大方,
>
> 各种类型的博客框架也都试过,jekyll,hexo等.
>
> 但是都因为各种原因,笔记本操作系统迁移,更换设备等最终很难持续进行下去
>
> 昨天(星期六, 2020年4月11日 8:51:36的前一天),我在刷题的时候,在jupyter notebook写了一些关于KMP算法内感想,想要理解一个算法,光靠代码真是不易,有图,或动图将事半功倍,看着我写在cell里的markdown笔记,以及他的导出html格式,我突然灵光一现! 这 jupyter notebook + github.io 不就是我要的blog编辑器,发布工具吗? 
>
> 我只需要写 jupyter notebook 然后导出一份对应的格式的html传到github里,不就是独一无二的,简单,适合迁移,没有多余挂件的个人博客了吗!
>
> 查了一下,目前还有没完美完整的解决方案.
>
> 存在的问题包括:
>
> - 渲染页面自定义(可以通过jinja2模板,配合nbconvert来解决)
> - 需要设计一个目录结构
> - 文件之间的关联问题
> - 等
>
> 解决上述问题,博客的构建至少也能够完成一般,虽然功能没有jekyll以及hexo全面
>
> 但是python的生态不必ruby,nodejs差多少,而且jupyter还支持40多种语言
>
> 这一项技术一定能够得到广泛的使用!

## 博客技术选型

选型要求:足够简单,方便后期迁移,满足基本写文需求不要太多花里胡哨功能,尽量减少拓展和外挂

## 博客使用说明书

1. 在wls环境下,进入vscode
2. 在_draft中编写草稿
3. 在_post中发布
4. 本地预览直接调用`jekyll serve`
